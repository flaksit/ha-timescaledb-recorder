<objective>
Wire both worker threads to the extended retry decorator and add the watchdog-context capture (`_last_exception`, `_last_context`) so a watchdog-spawned async task can read meaningful state off a dead thread after it exits.

Purpose: Every Phase 3 recovery path — stall notification, repair-issue create/clear, db_unreachable, watchdog restart — flows through attributes and hooks added by this plan. Without these wiring changes, the retry decorator and notifications.py are isolated pieces that nothing consumes.

Output: Both workers gain three hook methods (`_stall_hook`, `_recovery_hook`, `_sustained_fail_hook`), two attributes (`_last_exception`, `_last_context`), last-op tracking for each retried operation, and an outer try/except in `run()` that converts uncaught exceptions into "thread exit with context" rather than silent death. Test files gain coverage for each new hook and the outer-catch behaviour.

Implements: D-02 (per-worker stalled-issue bridge), D-03 (extended retry decorator consumed), D-06 (worker run() outer try/except + _last_exception/_last_context), D-11 (db_unreachable via sustained_fail hook).
</objective>
