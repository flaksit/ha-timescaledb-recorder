# Quick Task 260424-l5k: Fix backfill entity set for non-registry entities

## Goal

Entities without a `unique_id` (sun.sun, zone.home, conversation.*) are absent from
`entity_reg.entities` and thus silently excluded from backfill. Startup states are
never written to TimescaleDB for these entities. Fix by adding `hass.states.async_all()`
as a third source for the entity set in `backfill_orchestrator`.

## Tasks

### Task 1: Expand entity set in backfill_orchestrator

**File:** `custom_components/ha_timescaledb_recorder/backfill.py` lines 162-170

Add `state_machine_entities` from `hass.states.async_all()` (event-loop-safe call,
orchestrator runs in async context) and union into `entities`:

- `state_machine_entities = {state.entity_id for state in hass.states.async_all() if entity_filter(state.entity_id)}`
- `entities = live_entities | state_machine_entities | open_entities`
- Update comment (D-08-f) to document the third source.

**verify:** `entities` now includes entity_ids from state machine even when absent from entity_reg.

### Task 2: Update tests

**File:** `tests/test_backfill.py`

- In tests that check entity set construction: explicitly set `hass.states.async_all.return_value = []` where non-registry entities are irrelevant.
- Add new test `test_orchestrator_includes_non_registry_entities` verifying that an entity in `hass.states` but not in `entity_reg.entities` is included in the entity set passed to `get_significant_states`.

**verify:** All existing tests pass; new test passes; `hass.states.async_all` called once per backfill cycle.
