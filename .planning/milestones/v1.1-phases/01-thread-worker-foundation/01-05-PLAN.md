Refactor `syncer.py` to be a thin registry event relay. The four `@callback` handlers enqueue `MetaCommand` items instead of scheduling async tasks. The `_async_process_*_event` methods are deleted. The four change-detection helpers (`_entity/device/area/label_row_changed`) are converted from `async def` with asyncpg to sync `def` accepting a `psycopg.Cursor`.

The constructor accepts `queue: queue.Queue | None` (default `None`) to support this two-phase construction pattern.

Purpose: Registry param extraction is safe only in the event loop (`@callback` context). DB writes and change-detection reads must run in the worker thread. This plan moves the boundary correctly: extract params in event loop, execute SQL in worker.

Output: `syncer.py` where all `@callback` handlers enqueue `MetaCommand`; change-detection helpers are sync and use psycopg3 cursor; `bind_queue()` provides clean queue wiring API.

```python
async def async_start(self) -> None:
    """Iterate all four registries in the event loop and enqueue snapshot MetaCommands.

    D-04: snapshot items are enqueued BEFORE registering event listeners to ensure
    no registry events are missed and initial snapshot data is queued for the worker.

    Accepted race window (D-04 design decision): there is a small window between the
    snapshot iteration and listener registration where a registry change event could
    be missed. This is an accepted risk per D-04 — the snapshot + listener ordering
    minimises the window but does not eliminate it entirely. Phase 1 accepts this;
    Phase 3 (backfill) can close any gaps via periodic reconciliation if needed.
    """
    self._entity_reg = er.async_get(self._hass)
    self._device_reg = dr.async_get(self._hass)
    self._area_reg = ar.async_get(self._hass)
    self._label_reg = lr.async_get(self._hass)

    now = datetime.now(timezone.utc)

    for entry in self._entity_reg.entities.values():
        params = self._extract_entity_params(entry, now)
        self._queue.put_nowait(MetaCommand(
            registry="entity", action="create",
            registry_id=entry.entity_id, old_id=None, params=params,
        ))

    for entry in self._device_reg.devices.values():
        params = self._extract_device_params(entry, now)
        self._queue.put_nowait(MetaCommand(
            registry="device", action="create",
            registry_id=entry.id, old_id=None, params=params,
        ))

    for entry in self._area_reg.async_list_areas():
        params = self._extract_area_params(entry, now)
        self._queue.put_nowait(MetaCommand(
            registry="area", action="create",
            registry_id=entry.id, old_id=None, params=params,
        ))

    for entry in self._label_reg.async_list_labels():
        params = self._extract_label_params(entry, now)
        self._queue.put_nowait(MetaCommand(
            registry="label", action="create",
            registry_id=entry.label_id, old_id=None, params=params,
        ))

    # Register listeners AFTER enqueuing snapshot (D-04: no missed events)
    self._cancel_listeners.append(
        self._hass.bus.async_listen(
            EVENT_ENTITY_REGISTRY_UPDATED, self._handle_entity_registry_updated
        )
    )
    self._cancel_listeners.append(
        self._hass.bus.async_listen(
            EVENT_DEVICE_REGISTRY_UPDATED, self._handle_device_registry_updated
        )
    )
    self._cancel_listeners.append(
        self._hass.bus.async_listen(
            EVENT_AREA_REGISTRY_UPDATED, self._handle_area_registry_updated
        )
    )
    self._cancel_listeners.append(
        self._hass.bus.async_listen(
            EVENT_LABEL_REGISTRY_UPDATED, self._handle_label_registry_updated
        )
    )
```

```python
def _entity_row_changed(
    self, cur: psycopg.Cursor, entity_id: str, new_params: tuple
) -> bool:
    """Return True if the current open entity row differs from new_params.

    Receives a dict_row cursor from DbWorker — row["name"] access is valid.
    new_params indices: 0=entity_id, 1=ha_entity_uuid, 2=name, 3=domain,
    4=platform, 5=device_id, 6=area_id, 7=labels, 8=device_class,
    9=unit_of_measurement, 10=disabled_by, 11=valid_from, 12=extra
    """
    cur.execute(SELECT_ENTITY_CURRENT_SQL, (entity_id,))
    row = cur.fetchone()  # psycopg3: fetchone() not fetchrow() (asyncpg rename)
    if row is None:
        return True
    return (
        row["name"] != new_params[2]
        or row["platform"] != new_params[4]
        or row["device_id"] != new_params[5]
        or row["area_id"] != new_params[6]
        or sorted(row["labels"] or []) != sorted(new_params[7] or [])
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HA event loop → syncer @callback | Registry event data; entity_id values are HA-controlled |
| syncer @callback → worker queue | MetaCommand crosses thread boundary via queue.put_nowait() |
| worker thread → syncer change-detection helper | Cursor passed in; helper executes parameterized SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | _extract_*_params in @callback uses registry API | accept | Registry data comes from HA's in-memory registry, not external input; async_get is safe in event loop context per D-08 |
| T-05-02 | Denial of Service | put_nowait on reorder events | mitigate | _handle_area_registry_updated explicitly returns on action=="reorder" before enqueue |
| T-05-03 | Tampering | _extra_changed ignores modified_at | accept | Intentional design — _EXTRA_COMPARE_IGNORE excludes modified_at to prevent spurious SCD2 rows from HA internal registry updates |
| T-05-04 | Information Disclosure | Snapshot-then-register event-loss window | accept | D-04 accepted risk: the window is minimised by enqueuing snapshot before listener registration; Phase 3 backfill closes remaining gaps |
</threat_model>
